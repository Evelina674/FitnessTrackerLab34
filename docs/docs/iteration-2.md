# Бізнес-правила

1. Назва активності не може бути порожньою.
2. Тривалість активності повинна бути більшою за нуль.
3. Дистанція не може бути від’ємною.
4. Кількість підходів повинна бути більшою за нуль.
5. Кількість повторень повинна бути більшою за нуль.

# Використані патерни

Для створення активностей використано Factory Pattern.

Переваги:

- створення об’єктів винесено в окремий клас;
- легко додавати нові типи активностей;
- не потрібно змінювати існуючий код сервісів.

На Lab 37 можна буде додати нові типи активностей без зміни бізнес-логіки.
