# Iteration 1: Lab 34

## Що вже працює

- Консольний додаток дозволяє створити користувача, додати тренувальну сесію і переглянути підсумок.
- Всі доменні інваріанти перевіряються під час створення сутностей.
- Логіка зберігається в `FitnessTrackerService`, репозиторій реалізовано як `InMemoryFitnessSessionRepository`.
- Реалізовано вертикальний зріз: Console -> Service -> Domain -> Repository -> Console.
- Написано 5 юніт-тестів для ключових доменних правил і сервісів.

## Які сценарії будуть розширюватися на Lab 35

- Збереження профілів і тренувань у базі або файлі
- Аналіз прогресу за датами та фільтрація сесій
- Додавання нового типу активності і цілеутворення
- Інтеграційні тести для сервісу і репозиторія

## Ризики та невизначеності

- У першій ітерації дані зберігаються лише в пам'яті.
- Немає реалізації реєстрації кількох користувачів одночасно.
- Поточна консольна логіка підходить для одного завершеного сценарію, але потребує розширення для складніших use case.

## Підготовка до розширення

- `TrainingSession` і `Activity` спроєктовані для додавання нових типів активностей.
- `IFitnessSessionRepository` дозволяє замінити in-memory реалізацію на файл чи базу.
- `FitnessTrackerService` інкапсулює бізнес-правила окремо від UI.
